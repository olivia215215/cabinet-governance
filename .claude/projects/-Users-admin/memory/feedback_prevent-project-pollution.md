**铁律**：

1. **新任务先判归属**：接到任务，先判断是否属于当前目录已有项目。若无关，必须新建独立目录 + 独立 git 仓库，绝不往已有项目里塞。
2. **新建文件当即 commit**，不得延后。
3. **项目根目录 README 第一行写项目名**，防止脚手架命令跑错目录。
4. **不同项目分不同目录**，绝不在同一个 git 仓库里混。
5. **遇到 `git status` 出现大量 `??` 和 `D`**，先停手排查，不要盲目 checkout。

**Why:** 
- 天子日课项目被另一个 IT 资产管理系统污染——50+ 文件混入同一目录，原有 API 路由被删除、核心文件被覆盖。
- 另一次，用户给了全新任务（与天子日课无关），首辅未判归属直接塞进当前目录，导致两个项目互相覆盖。

**How to apply:**
- 收到任务第一反应不是"怎么写"，而是"这是哪个项目的活"
- 如果是全新项目：先在独立路径 `mkdir` + `git init`，再派大臣干活
- 如果不确定归属：问用户确认目录位置
